R10 — Auto-generate akun OPD + paksa ganti password + reset password
---------------------------------------------------------------
KONTRAK dengan frontend:
- POST /verifikator/opd (store) → redirect back dengan flash
  `generatedCredentials: { username, password }` (password plaintext hanya
  sekali di flash ini; frontend menampilkan dialog salin).
- POST /verifikator/opd/{opd}/reset-password → flash sama.
- users.must_change_password (boolean). Saat admin (OPD/verifikator) login
  dengan must_change_password=true → redirect ke halaman ganti password:
  render Inertia 'auth/force-password' (frontend saya buat), form
  POST /admin/password-baru { password, password_confirmation } →
  set password, must_change_password=false, lanjut ke dasbor.
  Middleware guard agar dasbor tidak bisa diakses sebelum ganti password.
- Welcome page: daftar OPD dirender dari prop `opds` (HomeController@index
  sudah kirim) — pastikan OPD baru is_active langsung ikut terkirim,
  sertakan field `inisial_opd` (lihat R11) dan `description`.
PROMPT:
"1) Migration: users.must_change_password boolean default false;
   users.last_login_at timestamp nullable (dipakai R12).
2) Verifikator\OpdController@store: setelah membuat OPD, buat juga akun
   User role admin_opd (username dari kode/inisial OPD, password acak
   Str::password(12)), kirim kembali via flash generatedCredentials.
3) Route + action reset password OPD (regenerate password acak, set
   must_change_password=true, flash generatedCredentials).
4) Middleware EnsureMustChangePassword (alias mis. password.changed):
   admin dengan must_change_password=true dipaksa ke auth/force-password;
   endpoint POST admin/password-baru memvalidasi & menyimpan.
5) Set users.last_login_at = now() pada setiap login sukses (Fortify
   authenticateUsing + OtpLoginController verify).
6) Tes: store OPD membuat user + flash kredensial; login pertama dipaksa
   ganti password; reset password bekerja."
