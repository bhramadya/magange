Berikut adalah pembaruan lengkap dan spesifik untuk **PRD (Product Requirements Document)** dan **Flowchart (Mermaid)** website E-Magang. Pembaruan ini telah mengintegrasikan semua hasil revisi, termasuk penambahan form, perombakan tugas admin, otomatisasi *Cron Job*, serta pemisahan keamanan *login*.

### 📄 PRODUCT REQUIREMENTS DOCUMENT (PRD) - REVISI FINAL

**1. Project Overview & Tech Stack**
*   **Nama Proyek:** Sistem Manajemen E-Magang Terpusat.
*   **Tech Stack:** Framework Laravel terbaru dikombinasikan dengan React/Inertia.js untuk antarmuka yang dinamis, serta PostgreSQL sebagai *database*.
*   **Tujuan:** Memfasilitasi pendaftaran magang tanpa registrasi awal, dengan sistem verifikasi dua tahap, penjadwalan otomatis (*Cron Job*), dan keamanan tinggi.

**2. User Roles & Permissions**
*   **User (Anak Magang):** Mengisi form magang, mengakses dasbor menggunakan OTP, melihat status, mengunggah laporan, mengisi survei, dan mengunduh sertifikat.
*   **Admin Verifikator:** Bertugas meninjau pengajuan masuk. Tidak lagi mengisi divisi/pembimbing, melainkan hanya **memberikan catatan khusus** untuk diteruskan ke Admin OPD. Memiliki akses mengubah ketersediaan kuota semua instansi dan bertugas mengunggah sertifikat selesai magang.
*   **Admin OPD:** Bertugas menerima lemparan pengajuan. Saat menyetujui, Admin OPD **wajib mengisi Divisi/Bidang, Pembimbing Lapangan, dan Penanggung Jawab**. Admin OPD hanya bisa mengubah ketersediaan kuota OPD-nya sendiri.

**3. UI/UX & Tampilan Frontend**
*   **Landing Page:** Menampilkan alur pendaftaran, FAQ, dan *card* OPD yang memuat informasi ketersediaan atau kuota magang.
*   **Formulir Publik:** Meminta data: Tujuan, Durasi, Kampus, Dosen, WA, Email. **Tambahan:** Kolom "Jurusan" (opsional) dan teks area "Keahlian/Keterampilan" diletakkan di atas kolom alamat.
*   **Card Pengajuan (Dasbor Verifikator):** Wajib menampilkan informasi "Keahlian" user agar admin dapat menilai kualifikasi peserta.
*   **Sidebar Admin:** Menu "Pengaturan" dihapus agar tidak membingungkan admin dalam mengubah data profil.

**4. Security & Authentication Logic**
*   **Pemisahan Alur Login:**
    *   **User:** Memasukkan Email, lalu menerima kode OTP. Tidak ada *password* statis.
    *   **Admin:** Menggunakan URL khusus (`/admin/login`) dengan form *Username* dan *Password* biasa tanpa OTP.
*   **Single Session Browser:** 1 *browser* tidak diizinkan *login* ke 2 akun berbeda secara bersamaan. Jika terjadi, sistem akan melempar *Error 403 Forbidden*.
*   **Proteksi Ekstra:** Wajib menyertakan anti-*Bruteforce*, *SQL/XSS Injection*, *Rate Limiting* (IP + Email), dan *Captcha* (model geser ala TikTok).

**5. Core Workflow & Otomatisasi (Cron Job)**
*   **Fase Verifikasi & Email Notifikasi:**
    *   Saat Admin OPD menekan "Setujui", harus muncul *alert*: *"Notif ini akan dikirim ke peserta magang. Peserta akan datang berkunjung ke kantor setelah diterima pengajuan ini"*.
    *   Sistem mengirim email persetujuan dengan *header* biru besar **"magang kota Madiun"** dan teks **"disetujui"**. Email memuat detail: asal, durasi, divisi, pembimbing, penanggung jawab, catatan, dan instruksi kedatangan ke kantor. Status tiket berubah menjadi **"Disetujui"**.
*   **Otomatisasi Cron Job (Harian):**
    *   *Cron Job* berjalan tiap jam 1 pagi.
    *   Jika status **"Disetujui"** dan periode magang dimulai hari ini -> ubah ke **"Sedang Magang"**.
    *   Jika status **"Sedang Magang"** dan periode magang berakhir hari ini -> ubah ke **"Selesai Magang"**.
*   **Filter Dasbor:** Urutan *tab* filter: *Perlu Keputusan, Disetujui, Sedang Magang, Selesai Magang, Ditolak, Semua*.
*   **Penyelesaian Magang (4 Aktor):** Status magang menjadi "Selesai" dapat dipicu oleh 4 pihak: 1) Sistem (*Cron Job*), 2) Admin Verifikator, 3) Admin OPD, dan 4) User (upload laporan). Setelah selesai, Admin Verifikator mengunggah surat sertifikat. User wajib mengisi survei kepuasan agar tombol unduh sertifikat terbuka.

    %% FASE 1: Pendaftaran
    subgraph Fase1 [Fase 1: Pendaftaran & Lolos Keamanan]
        A([Mulai: User Buka Landing Page]) --> B(Isi Form: Tujuan, Durasi, Kampus, Email, WA)
        B --> B1(Isi Form Tambahan: Jurusan & Keahlian)
        B1 --> C{Lolos Captcha reCAPTCHA & Rate Limit?}
        C -- Tidak --> D[Akses Diblokir / Error]
        C -- Ya --> E{Cek Email/WA di Database}
        E -- Belum Ada --> F[Sistem Buat Akun Baru]
        E -- Sudah Ada --> G[Gunakan Akun Terdaftar]
        F --> H
        G --> H
    end

    %% FASE 2: Verifikasi Revisi
    subgraph Fase2 [Fase 2: Verifikasi Bertingkat & Tugas Baru]
        H[Pengajuan Masuk Dasbor Admin Verifikator] --> I{Tinjau Pengajuan & Keahlian}
        I -- Tidak Valid --> J[Tolak & Kirim Email Penolakan]
        I -- Valid --> K[Verifikator Beri Catatan Khusus & Lempar ke OPD]
        K --> L{Admin OPD: Putuskan}
        L -- Tolak --> M[Tolak & Kirim Email Penolakan]
        L -- ACC --> N[Muncul Alert Peringatan Kedatangan]
        N --> O[Admin OPD Isi: Divisi, Pembimbing, Penanggung Jawab]
        O --> P[Kirim Email ACC Detail + Header Biru 'Magang Kota Madiun']
        P --> Q[Update Status: Disetujui]
    end

    %% FASE 3: Login Terpisah & Keamanan
    subgraph Fase3 [Fase 3: Akses Login Terpisah]
        %% Alur User
        R([User Buka URL Utama]) --> S(Input Email)
        S --> T[Kirim OTP via Email]
        T --> U(User Masukkan OTP)
        U --> V{Apakah Browser Sedang Digunakan Akun Lain?}
        V -- Ya --> W[Tolak Akses: Error 403 Forbidden]
        V -- Tidak --> X[Berhasil Masuk Dasbor User]

        %% Alur Admin
        Y([Admin Buka URL /admin/login]) --> Z(Input Username & Password Biasa)
        Z --> AA[Berhasil Masuk Dasbor Admin]
    end

    %% FASE 4: Cron Job & 4 Aktor Selesai
    subgraph Fase4 [Fase 4: Cron Job Otomatisasi & Penyelesaian]
        Q -.-> AB{Cron Job: Cek Periode Harian jam 01.00}
        AB -- Tanggal Mulai Tiba --> AC[Update Status: Sedang Magang]
        AC -.-> AD{Kondisi Penyelesaian Terpenuhi?}

        %% 4 Aktor Penyelesaian
        AD -- 1. Sistem Cron Job: Tanggal Berakhir Tiba --> AE[Update Status: Selesai Magang]
        AD -- 2. Dieksekusi Admin Verifikator --> AE
        AD -- 3. Dieksekusi Admin OPD --> AE
        X -.-> AF(4. User Upload Laporan & Klik Selesai)
        AF --> AE

        AE --> AG[Admin Verifikator Upload Surat Sertifikat]
        AG --> AH[Sertifikat Masuk Dasbor User - Tombol Terkunci]
        AH --> AI(User Isi Survei Kepuasan & Rating)
        AI --> AJ[Tombol Download Terbuka]
        AJ --> AK([Selesai: User Download Sertifikat])
    end
