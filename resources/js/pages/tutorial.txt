--------------------------------------------------------------------------------
 CARA A — Checkout seluruh branch newback (paling sederhana untuk "lihat semua")
--------------------------------------------------------------------------------
Membuat branch lokal `newback` yang melacak remote, lalu pindah ke sana.
Kerugian: kamu MENINGGALKAN branch `front` (pekerjaanmu sekarang). Pastikan
`front` sudah commit/bersih dulu.

    git switch front            # pastikan di front & bersih dulu
    git status                  # harus "nothing to commit, working tree clean"
    git switch -c newback origin/newback   # buat + pindah ke newback
    # ...lihat/jalankan kodenya...
    git switch front            # balik ke front saat selesai

--------------------------------------------------------------------------------
 CARA C — Cherry-pick commit-nya ke branch sekarang (bawa 1 commit utuh)
--------------------------------------------------------------------------------
Menerapkan SEMUA perubahan commit 7c798ac sebagai commit baru di branch aktif.
Ini yang paling mungkin menimbulkan KONFLIK (karena file yang sama diubah di front).

    git switch front
    git cherry-pick 7c798ac
    # kalau konflik -> selesaikan (lihat bagian MENANGANI KONFLIK), lalu:
    git cherry-pick --continue
    # kalau mau batal total:
    git cherry-pick --abort

--------------------------------------------------------------------------------
 CARA D — Terapkan sebagai PATCH (git am / git apply)
--------------------------------------------------------------------------------
Pakai file patch yang sudah dibuat.

    # metode 1: git am (mempertahankan pesan & author commit asli)
    git am 0001-UWES-ALL.patch
    # kalau gagal/konflik:
    git am --abort            # batal
    #   atau selesaikan konflik lalu:  git am --continue

    # metode 2: git apply (hanya menaruh perubahan ke working tree, tanpa commit)
    git apply --3way UWES_ALL_7c798ac.patch
    #   --3way membantu auto-merge; kalau bentrok akan muncul penanda konflik.
    #   Cek dulu tanpa mengubah apa pun:
    git apply --check UWES_ALL_7c798ac.patch

--------------------------------------------------------------------------------
 CARA E — Inspeksi tanpa ganggu branch aktif (git worktree) [opsional, aman]
--------------------------------------------------------------------------------
Membuka commit itu di FOLDER TERPISAH, jadi branch `front` tak tersentuh.

    # sekarang ada salinan repo di folder ../magange-newback berisi kode commit itu
    cd ../magange-newback        # jelajahi / jalankan

================================================================================
 MENANGANI KONFLIK (kalau muncul saat Cara C / D)
================================================================================
Konflik ditandai blok berikut di dalam file:

    <<<<<<< HEAD
    (kode versi branch front — punyamu sekarang)
    =======
    (kode versi dari commit 7c798ac — UWES ALL)
    >>>>>>> 7c798ac

Langkah:
    1. Buka tiap file yang konflik (git status akan menandainya "both modified").
    2. Edit manual: pilih salah satu sisi, atau gabungkan keduanya, lalu HAPUS
       ketiga baris penanda (<<<<<<<, =======, >>>>>>>).
    3. Tandai selesai:      git add <file>
    4. Lanjutkan:
         - kalau cherry-pick:  git cherry-pick --continue
         - kalau git am:       git am --continue
    5. Kalau mau membatalkan semuanya & kembali ke kondisi semula:
         - cherry-pick:  git cherry-pick --abort
         - git am:       git am --abort
         - merge:        git merge --abort

Tips melihat 3 versi saat konflik:
    git checkout --ours   <file>   # ambil versi front
    git checkout --theirs <file>   # ambil versi 7c798ac
    (lalu tetap: git add <file>)
